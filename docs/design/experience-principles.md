# Experience Principles · עקרונות UX מחייבים

עקרונות שכל מסך, Flow ו-Feature חייבים לעמוד בהם. אלה אינם המלצות — הם תנאי מעבר ב-Design Review.

## 10 העקרונות

1. **עד 2–3 קליקים לכל פעולה חשובה.** אם פעולה דורשת יותר — זה באג בעיצוב.
2. **אין Dead Ends.** לכל מסך יש "מה הלאה". כל מבוי סתום מקבל פעולה או דרך חזרה.
3. **כל מסך מסביר את הצעד הבא.** המשתמש לעולם לא תוהה "ומה עכשיו?".
4. **כל Error מרגיע.** שגיאה מורידה מתח, מסבירה, ומציעה פתרון. ([brand](brand-guidelines.md))
5. **כל Success משמח.** אישור חם וברור לכל פעולה שהצליחה.
6. **כל Empty State מדריך.** ריק = הזמנה + צעד ראשון, לא "אין נתונים".
7. **כל Dashboard נותן תחושת שליטה.** המשתמש רואה במבט אחד איפה הוא עומד ומה דחוף.
8. **כל Wizard מפחית לחץ.** צעד-אחר-צעד, התקדמות גלויה, אפשר לחזור, אי אפשר לטעות.
9. **כל Flow פשוט יותר ממה שהמשתמש מצפה.** ברירות מחדל חכמות, פחות שדות, פחות החלטות.
10. **כל Feature חוסך לזוג זמן.** אם זה לא חוסך זמן או מתח — לא בונים אותו.

## עקרונות תומכים

- **Progressive disclosure** — מראים את העיקר; מורכבות נחשפת רק כשצריך.
- **Forgiving** — ביטול (undo) על פעולות הרסניות; אישור על מה שלא הפיך.
- **State always visible** — Loading / Empty / Error / Success מטופלים בכל מסך, לא רק ה-"happy path".
- **One primary action per screen** — כפתור Primary אחד דומיננטי; השאר משניים.
- **Recognition over recall** — המשתמש בוחר מרשימה, לא זוכר בעל-פה.

## בדיקת "וואו" (Done = עובר את כל אלה)

- [ ] הפעולה המרכזית במסך מזוהה תוך 3 שניות.
- [ ] כל ארבעת המצבים (Loading/Empty/Error/Success) מעוצבים.
- [ ] אין מסך שמסתיים בלי "מה הלאה".
- [ ] באזור הזוג: כל פעולה ניתנת לביצוע ביד אחת עם האגודל.
- [ ] טקסט אחד לפחות גורם למשתמש לחייך / להירגע.
