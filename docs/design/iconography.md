# Iconography · שפת אייקונים

## ההחלטה: lucide-react (אחד בלבד)

המוצר משתמש כבר ב-[`lucide-react`](https://lucide.dev) ברוב המסכים. זו שפת האייקונים **היחידה** מכאן ואילך.
אסור לערבב: ❌ Material Symbols, ❌ emoji-as-icon בממשק ניהול, ❌ אייקונים מצוירים ידנית, ❌ ספריות אחרות.

> הערה: עיצובי Stitch מגיעים עם Material Symbols. בעת מימוש pixel-accurate — ממירים לאייקון lucide המקביל הקרוב ביותר. זו סטייה מאושרת ומכוונת (עקביות > התאמה מילולית של ספריית אייקונים).

## כללים

| נושא | כלל |
|------|-----|
| **גדלים** | 16 (inline/קטן), 18 (badges), 20 (nav/כפתורים), 22–24 (כותרות/KPI), 28+ (אמפטי/הירו) |
| **Stroke** | ברירת מחדל lucide (2). דק יותר (1.5) למסכי הזוג העדינים; 3 רק ל-check בתוך checkbox |
| **צבע** | יורש מהטקסט (`currentColor`). מבטא זהב/זית רק כשהאייקון הוא הנושא |
| **מרווח** | `gap: 6–8px` בין אייקון לטקסט |
| **יישור** | תמיד `align-items: center`. אייקוני כיוון (חצים) — ממורכזים ל-RTL |

## RTL — היפוך אייקוני כיוון

בממשק RTL, אייקונים שמסמנים "קדימה/הבא" מצביעים **שמאלה**:
- "הבא / המשך / שלח" → `ArrowLeft`
- "חזור / הקודם" → `ArrowRight`
- Chevron בתפריטים נפתחים — בודקים ויזואלית שהכיוון נכון ב-RTL.

## עקביות

- אותו מושג = אותו אייקון בכל המערכת. אורחים תמיד `Users`, אירוע תמיד `Calendar`, וכו'.
- מתחזקים מילון אייקון↔מושג בעת הוספת מסכים חדשים.
