# Design Sprint Mode Policy
## רגע לפני AI Company OS | CEO Directive | 2026-06-26 | IMMUTABLE

---

## The Golden Rule

```
No new frontend implementation.
No UI refactoring.
No redesign implementation.
No CSS work.
No component creation.
```

**Until the complete product design has been approved.**

The only exceptions:
1. Backend work (APIs, DB, infrastructure)
2. Critical production bug fixes (P0/P1 incidents only)

---

## Why

We are designing one coherent product.

Not one screen at a time.
Not one route at a time.
Not one component at a time.

Implementation that begins before the full design is approved
produces incoherence — screens that don't speak the same visual language,
patterns that don't reuse the design system,
decisions that have to be reversed.

Design first. Build once. Build correctly.

---

## The Implementation Gate

Implementation begins only after:

```
✓ Every Experience has an approved Design Pack
✓ The complete product has a consistent design language
✓ The Design Library reflects all approved patterns
✓ CEO approves the Product Design Master Plan
```

Only then: create an Implementation Roadmap.
Only then: begin implementation.

---

## Execution Order

### Phase 1 — Master Plan ✅
Audit the entire product. Map all experiences. Document business goals, journeys, emotions, metrics.

### Phase 2 — Design Packs (per Experience, in priority order)
For each experience: journey map + all screens + all states + 3 Stitch prompts.

### Phase 3 — Stitch Design (per Experience)
Send complete experience briefs to Stitch. Never single isolated screens.

### Phase 4 — Design Review (per Experience)
All 16 criteria. All 7 lenses. Decision Log. Memory. Library. Intelligence Report.

### Phase 5 — CEO Approval (per Experience)
Present each experience's designs to CEO. Wait for explicit approval.

### Phase 6 — Implementation Roadmap
After all experiences approved: sequence by impact + dependency. Define milestones.

### Phase 7 — Implementation (experience by experience)
Build in the order defined by the Roadmap. Not before.

---

## Stitch Brief Format (Required for Every Submission)

Every Stitch brief must include:

```
Context: [user journey before this experience]
Experience: [name and description]
User: [who they are, what they just did]
Goal: [what they need to accomplish]
Emotion: [what they should feel at each step]
Screens: [complete list with ALL states]
Brand: ivory:#FDFAF5 cream:#F6F1E8 gold:#C5A46D olive:#6B7B5A dark:#1C1008
       Frank Ruhl Libre (headings) + Heebo (body) | RTL | Mobile first
After: [where the user goes after this experience]
```

Never ask Stitch to design a single screen without the complete journey context.
