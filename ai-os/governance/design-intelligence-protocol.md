# Design Intelligence & Design Memory Protocol v1.0
## רגע לפני AI Company OS | CEO Directive | 2026-06-26 | IMMUTABLE

---

## The Law

**Every completed design must leave the company smarter than before.**

A screen is never considered complete until its knowledge has been captured
and shared across the AI Company OS.

Design knowledge is a strategic company asset — not a side effect of building.

---

## The New Wave Workflow

```
Research
    ↓
Experience Pack
    ↓
Design Pack
    ↓
Stitch Design
    ↓
Design Review                    ← 7 lenses (see below)
    ↓
Design Decision Log              ← mandatory before implementation
    ↓
Design Memory Update             ← patterns, emotions, discoveries
    ↓
Design Library Update            ← approved patterns become reusable standards
    ↓
Experience Intelligence Report   ← what the experience teaches us
    ↓
Company Brain Update             ← strategic design knowledge
    ↓
CEO Approval
    ↓
Implementation Plan
    ↓
Implementation
```

**Implementation is NOT allowed until all knowledge steps above it are complete.**

## Step 1 — Design Review (7 Lenses)

Review every screen through all 7 roles simultaneously:

| Role | What They Ask |
|------|--------------|
| Product Manager | Does this serve the business objective? |
| UX Director | Is the user journey clear and frictionless? |
| Design Director | Is this premium? Consistent? Emotionally resonant? |
| Customer Success Manager | Will users understand this without help? |
| Wedding Planner | Does this feel like it belongs at a wedding? |
| Accessibility Specialist | Is it usable by everyone? |
| Mobile UX Expert | Is this truly mobile-first? RTL-correct? |

### Review Criteria (all required)

```
[ ] Premium Feel
[ ] Trust
[ ] Emotion
[ ] Visual Hierarchy
[ ] Brand Consistency
[ ] Mobile Experience
[ ] Desktop Experience
[ ] Accessibility
[ ] RTL
[ ] Animations
[ ] Micro-interactions
[ ] CTA Clarity
[ ] White Space
[ ] Consistency
[ ] User Flow
[ ] Business Alignment
```

## Step 2 — Design Decision Log

**Location:** `ai-os/design/decisions/[wave-name]-decisions.md`

For every significant design decision:

```
Decision: [what was chosen]
Rationale: [why it was chosen]
Alternatives Considered: [what else was evaluated]
Rejection Reason: [why alternatives were rejected]
Business Objective: [which goal it serves]
User Problem Solved: [what friction it removes]
Emotion Created: [what the user feels]
```

Examples of decisions that require a log entry:

- Why gold instead of green?
- Why large typography instead of small?
- Why this spacing?
- Why this card hierarchy?
- Why illustration instead of photography?
- Why animation instead of static?
- Why one CTA instead of two?

---

## Step 3 — Design Memory

**Location:** `ai-os/design/memory/design-memory.md`

A living document. Updated after every Design Review.

Captures patterns discovered in user testing, design review, and post-release observation.

### Format

```
## [Pattern Category]

### Works Well
- [observation] — discovered in [wave/date]

### Avoid
- [observation] — discovered in [wave/date]
```

**This memory is automatically referenced in every future Design Review.**
No pattern should be re-discovered. Every lesson is permanent.

---

## Step 4 — Design Library

**Location:** `ai-os/design/library/`

Every approved screen contributes patterns. Organized by type:

```
layout-patterns.md       — page structure, grid, composition
interaction-patterns.md  — tap, swipe, press, long-press behaviors
animation-patterns.md    — transitions, micro-animations, loading
navigation-patterns.md   — flow between screens, back behavior
card-patterns.md         — card anatomy, hierarchy, content
form-patterns.md         — input fields, validation, submission
confirmation-patterns.md — success, confirmation, celebration moments
loading-patterns.md      — skeleton screens, spinners, progress
empty-state-patterns.md  — first-use, no-data, no-results states
error-patterns.md        — error, invalid, recovery states
success-patterns.md      — completion, achievement, delight moments
typography-patterns.md   — heading scales, body text, label text
color-patterns.md        — usage rules per color token
spacing-patterns.md      — consistent spacing decisions
```

**Future screens MUST reuse existing library patterns before inventing new ones.**
The library grows. It never shrinks.

## Step 6 — Company Brain Update

**Location:** `ai-os/brain/`

The Company Brain is the AI OS's long-term memory.

Design knowledge belongs in:

```
ai-os/brain/design-standards.md   — what we know about great design
ai-os/brain/product-principles.md — what we know about our users
ai-os/brain/visual-language.md    — the permanent visual decisions
```

After every wave:
- Update the relevant brain files
- Mark discoveries as "Validated" or "Hypothesis"
- Note which wave produced the insight

## The Implementation Gate

```
✓ Design Review completed (all 16 criteria)
✓ Design Decision Log created (all significant decisions documented)
✓ Design Memory updated (patterns added)
✓ Design Library updated (new patterns documented)
✓ Experience Intelligence report completed
✓ CEO approved the design
```

**All 6 must be ✓ before Implementation Plan is written.**

---

## Final Principle

> "Every decision, every review, every approved screen,
> and every lesson learned must become reusable knowledge
> for all future product development."

The Design Library, Design Memory, and Company Brain together
become the single source of truth for every future design decision.

Speed is not the goal. Intelligence is.
